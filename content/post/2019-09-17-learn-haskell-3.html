A recent relevation I had is: all I’m doing when I program is just playing with Product and Sum types. What are them?

tuple is the canonical example of the Product type:

:t (1 :: Int, "two")
## (1 :: Int, "two") :: (Int, [Char])

the name Product in product type might come from set theory: if we think of integers as one set and strings as another, then we can think of a tuple of type (Int, String) as a cartesian product of integer set and string set.

We can see this in other languages as well, for example, in scala’s type hierachy, Tuples inherits from Product.

Either is the canonical Sum type (a.k.a Coproduct).

data Either a b = Left a | Right b
